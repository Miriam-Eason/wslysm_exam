# 智慧题库与智能练测系统 — 技术路线

> **数据模型以《数据模型总表》为准。**

## 一、技术架构概览

### 1.1 技术选型

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 前端框架 | Next.js (App Router) + React | 全栈框架，前后端统一项目 |
| UI 组件库 | shadcn/ui + Tailwind CSS | 响应式布局，适配教师端（PC）与学生端（移动端） |
| 后端接口 | Next.js API Routes / Server Actions | 无需独立后端服务 |
| ORM | Prisma | 类型安全的数据库操作，自动迁移管理 |
| 数据库 | MySQL 8.0 | 关系型数据库，适合题目-试卷-用户-成绩的关联结构 |
| 部署平台 | 国内云服务器（阿里云/腾讯云轻量应用服务器） | 保障国内访问速度与数据合规 |
| 容器化 | Docker + Docker Compose | 应用与数据库分离部署，一键启停 |
| 版本管理 | Git + GitHub | 代码托管与版本控制 |

### 1.2 架构设计说明

本系统采用 Next.js 全栈架构，前端页面（React 组件）与后端接口（API Routes）统一在同一个项目中。教师端与学生端共享同一套代码库，通过用户角色与路由中间件进行权限隔离，无需拆分为独立项目或配置子域名。

路由规划如下：

- `/teacher/*` — 教师端功能（题库管理、组卷、成绩查看）
- `/student/*` — 学生端功能（刷题、复习、成绩查询）
- `/api/*` — 后端数据接口
- 中间件（Middleware）统一拦截未授权请求并跳转至登录页

学生端设计为移动优先的响应式布局，可配置为 PWA（渐进式 Web 应用），在手机浏览器中获得接近原生 App 的体验。

### 1.3 系统用户规模预估

| 指标 | 数值 |
|------|------|
| 注册用户总数峰值 | ≤ 600 人 |
| 同时在线峰值 | ≤ 200 人 |
| 教师用户数 | 约 10–20 人 |

上述规模下，单台 2 核 2G 轻量云服务器即可满足性能需求。考试高峰并发下 MySQL 8 + Next.js 同机内存偏紧，建议起步 4G。

## 二、开发环境搭建

### 2.1 本地环境要求

| 工具 | 版本要求 | 用途 |
|------|---------|------|
| Node.js | ≥ 18 | Next.js 运行环境 |
| npm / pnpm | 最新稳定版 | 包管理 |
| MySQL | 8.0 | 本地开发数据库 |
| Git | 最新稳定版 | 版本管理 |
| Docker Desktop | 最新稳定版 | 本地构建测试 Docker 镜像（可选） |
| Claude Code | 最新版 | AI 辅助编码 |

### 2.2 数据库初始化

本地仅需创建一个空数据库，不需要手动建表。表结构由 Prisma ORM 通过 schema 文件自动生成。

```sql
mysql -u root -p
CREATE DATABASE exam_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.3 项目初始化

```bash
# 创建 Next.js 项目
npx create-next-app@latest exam-system --typescript --tailwind --app

# 安装核心依赖
cd exam-system
npm install prisma @prisma/client
npm install -D prisma

# 初始化 Prisma
npx prisma init
```

初始化后在项目根目录生成 `prisma/schema.prisma` 文件和 `.env` 环境变量文件。

### 2.4 环境变量配置

项目根目录 `.env` 文件（不纳入 Git 版本管理）：

```env
DATABASE_URL="mysql://root:yourpassword@localhost:3306/exam_system"
NEXTAUTH_SECRET="随机生成的密钥"
```

不同环境（本地开发、云服务器）各自维护独立的 `.env` 文件，确保数据库连接指向正确的实例。

### 2.5 数据库表结构管理

所有表结构定义在 `prisma/schema.prisma` 文件中。最终设计采用**独立的 Teacher 与 Student 两张表**（不使用统一 User 表 + Role 枚举），以下为两个模型的骨架示例：

```prisma
datasource db {
  provider = "mysql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

model Teacher {
  id           Int      @id @default(autoincrement())
  username     String   @unique          // 登录账号
  passwordHash String                    // bcrypt 哈希，绝不存明文
  name         String
  createdAt    DateTime @default(now())
  // ... 关联关系（Class、QuestionBank、Exam）
}

model Student {
  id           Int      @id @default(autoincrement())
  studentNo    String   @unique @db.VarChar(10) // 9-10位学号，全局唯一身份
  name         String
  passwordHash String                            // bcrypt 哈希
  createdAt    DateTime @default(now())
  // ... 关联关系（Enrollment、Attempt）
}
```

> 完整模型见《数据模型总表》。学生以学号（studentNo）、教师以账号（username）分别作为登录凭据，认证时需对两张身份表各自实现 authorize 逻辑（详见第 2.4 节说明及下方认证说明）。

学生(学号)与教师(账号)为两张独立身份表，若使用 NextAuth 需采用 Credentials Provider，对两类身份分别实现 `authorize` 回调，不能共用同一套逻辑。

表结构变更流程：

```bash
# 修改 schema.prisma 后执行迁移
npx prisma migrate dev --name 描述本次变更

# 该命令自动完成：
# 1. 对比 schema 变更，生成 SQL 迁移脚本
# 2. 在本地 MySQL 中执行建表/改表
# 3. 重新生成 Prisma Client 类型代码
```

### 2.6 测试数据填充

编写 `prisma/seed.ts` 脚本，用于向数据库灌入初始测试数据（模拟教师账号、学生账号、示例题目、示例试卷等）。每次重置数据库后可一键恢复：

```bash
npx prisma db seed
```

## 四、部署架构

### 4.1 部署方案选型依据

| 方案 | 国内访问速度 | 部署便捷度 | 成本 | 长期可靠性 | 是否采用 |
|------|------------|-----------|------|-----------|---------|
| Vercel | 慢，偶尔不稳定 | 极简 | 免费起步 | 高 | 否（大陆无节点） |
| Zeabur | 中等 | 接近 Vercel | ≈35 元/月 | 不确定 | 否（节点不在大陆） |
| 国内轻量云服务器 | 快，最稳定 | 需初始配置 | 50–80 元/月 | 高 | **采用** |

最终方案：**国内云服务器 + Docker Compose（Next.js 容器 + MySQL 容器）**。

### 4.2 服务器配置建议

| 配置项 | 推荐值 |
|-------|--------|
| 云服务商 | 阿里云/腾讯云轻量应用服务器 |
| CPU | 2 核 |
| 内存 | 2 GB（4 GB 更佳，考试高峰并发下 MySQL 8 + Next.js 同机内存偏紧，建议起步 4G） |
| 系统盘 | 50 GB SSD |
| 操作系统 | Ubuntu 22.04 / 24.04 LTS |
| 带宽 | 5 Mbps |
| 预估月费 | 50–80 元 |

### 4.3 容器架构

系统通过 Docker Compose 编排三个正式服务：**nginx**（入口反向代理）、**app**（Next.js 应用）、**db**（MySQL 数据库），统一由 compose 管理，通过 Docker 内部网络通信：

```
┌──────────────────────────────────────────────────────┐
│                  云服务器 (Ubuntu)                     │
│                                                      │
│  ┌──────────────────────────────────────────────┐    │
│  │   nginx 容器（正式服务，compose 统一编排）     │    │
│  │   nginx:alpine                               │    │
│  │   端口: 80 (HTTP) / 443 (HTTPS)              │    │
│  │   反向代理 → app:3000                        │    │
│  └──────────────────┬───────────────────────────┘    │
│                     │                                │
│  ┌──────────────────▼──────┐  ┌──────────────────┐  │
│  │   app 容器               │  │   db 容器         │  │
│  │   Next.js (standalone)  │──│   MySQL 8.0      │  │
│  │   内部端口: 3000         │  │   端口: 3306     │  │
│  └─────────────────────────┘  └────────┬─────────┘  │
│                                        │             │
│                                  mysql_data 卷       │
│                                 (持久化存储到宿主机)  │
└──────────────────────────────────────────────────────┘
```

关键设计原则：

- 应用容器与数据库容器分离，生命周期独立。更新应用代码只需重建 app 容器，MySQL 容器和数据不受影响。
- MySQL 数据通过 Docker Volume 持久化到宿主机磁盘，容器删除或重建不会丢失数据。
- **Nginx 已作为正式服务纳入 compose**，统一负责 80/443 端口入口与反向代理到 app:3000，生产环境由 Nginx 作为唯一外部入口，app 容器不对宿主机直接暴露 3000 端口。

### 4.4 核心配置文件

**next.config.js（启用 standalone 输出）：**

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'standalone',
}

module.exports = nextConfig
```

**Dockerfile：**

```dockerfile
# === 构建阶段 ===
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npx prisma generate
RUN npm run build

# === 运行阶段 ===
FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production

COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public
COPY --from=builder /app/node_modules/.prisma ./node_modules/.prisma
COPY --from=builder /app/prisma ./prisma

EXPOSE 3000
CMD ["node", "server.js"]
```

**docker-compose.yml：**

```yaml
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d:ro
      - ./nginx/certs:/etc/nginx/certs:ro
    depends_on:
      - app
    restart: unless-stopped

  app:
    build: .
    # 生产环境由 nginx 统一入口，app 不对宿主机暴露端口；
    # 如需本地调试可临时加回 "3000:3000"
    expose:
      - "3000"
    environment:
      - DATABASE_URL=mysql://examuser:yourpassword@db:3306/exam_system
    depends_on:
      db:
        condition: service_healthy
    restart: unless-stopped

  db:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=rootpassword
      - MYSQL_DATABASE=exam_system
      - MYSQL_USER=examuser
      - MYSQL_PASSWORD=yourpassword
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - "3306:3306"
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 5s
      retries: 10
    restart: unless-stopped

volumes:
  mysql_data:
```

**配置说明：**

- `depends_on` + `healthcheck`：确保 MySQL 完全启动后 Next.js 才开始连接，避免启动时连接失败。
- `volumes: mysql_data`：将 MySQL 数据持久化到宿主机，容器生命周期与数据生命周期解耦。
- `restart: unless-stopped`：服务器重启后容器自动恢复运行。
- **Nginx 反向代理与 HTTPS 证书**：`./nginx/conf.d/` 目录中放置站点配置文件，将 `server_name` 指向域名，`proxy_pass http://app:3000` 反代到应用容器。SSL 证书由 Let's Encrypt（certbot）申请后放入 `./nginx/certs/`，certbot 支持自动续期（`certbot renew`），挂载为只读卷以保证配置安全。HTTPS 就绪后，Nginx 可同时配置 HTTP→HTTPS 301 重定向。

## 五、部署操作手册

### 5.1 服务器初始化（一次性操作）

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# 重新登录使 docker 组生效

# 2. 安装 Docker Compose（Docker 20.10+ 已内置）
docker compose version

# 3. 安装 Git
sudo apt update && sudo apt install -y git

# 4. 克隆项目代码
git clone https://github.com/yourname/exam-system.git
cd exam-system
```

### 5.2 首次部署

```bash
# 1. 启动所有服务（首次会自动构建镜像，耗时约 2-5 分钟）
docker compose up -d --build

# 2. 等待 MySQL 完全启动后，执行数据库迁移（建表）
docker compose exec app npx prisma migrate deploy

# 3. 灌入初始数据（管理员账号、示例题目等）
docker compose exec app npx prisma db seed

# 4. 验证服务运行状态
docker compose ps

# 5. 浏览器访问 http://服务器IP:3000 验证
```

### 5.3 日常更新流程

```bash
# 在服务器上执行
cd exam-system
git pull
docker compose up -d --build app   # 仅重建应用容器，数据库不受影响

# 如果本次更新包含数据库表结构变更，额外执行：
docker compose exec app npx prisma migrate deploy
```

整个更新过程约 1–2 分钟，期间服务短暂不可用。

### 5.4 数据备份策略

配置定时自动备份（通过 crontab）：

```bash
# 编辑定时任务
crontab -e

# 添加以下内容：每天凌晨 3 点自动备份数据库
0 3 * * * docker compose -f /home/ubuntu/exam-system/docker-compose.yml exec -T db mysqldump -u examuser -p'yourpassword' exam_system > /home/ubuntu/backup/exam_$(date +\%Y\%m\%d).sql

# 可选：定期清理 30 天前的旧备份
0 4 * * * find /home/ubuntu/backup/ -name "exam_*.sql" -mtime +30 -delete
```

手动备份：

```bash
docker compose exec -T db mysqldump -u examuser -p'yourpassword' exam_system > backup.sql
```

数据恢复：

```bash
docker compose exec -T db mysql -u examuser -p'yourpassword' exam_system < backup.sql
```

### 5.5 域名绑定与 HTTPS

Nginx 已正式纳入 docker-compose（见第 4.4 节），无需临时添加容器，直接配置反代与证书即可：

1. 购买域名并完成 ICP 备案（国内服务器必须备案）
2. 域名 DNS 解析指向服务器 IP
3. 在 `./nginx/conf.d/` 中编写站点配置，将 `proxy_pass` 指向 `http://app:3000`
4. 使用 Let's Encrypt 免费 SSL 证书（通过 certbot 申请，证书文件放入 `./nginx/certs/`，certbot 支持自动续期）
5. `docker compose restart nginx` 使配置生效，无需重建其他容器

## 七、技术风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 服务器宕机 | 服务中断 | 配置容器自动重启（restart: unless-stopped）；数据已持久化不会丢失 |
| 数据丢失 | 题库和成绩数据不可恢复 | 每日自动备份 + 定期下载备份文件到本地 |
| 并发访问导致响应慢 | 考试高峰期体验差 | 2 核 2G 可支撑 200 并发；如不足可升级为 4 核 4G |
| 数据库迁移失败 | 表结构更新中断 | Prisma 迁移支持回滚；先在本地验证迁移脚本再部署 |
| 后续需要数据库切换 | 从 MySQL 迁移到 PostgreSQL | Prisma 支持切换数据库，仅需修改 schema 中的 provider 配置 |
