# 数据模型总表（权威 Schema · 单一事实来源）

> 本文档合并《项目功能设计》《学生导入方案设计》《题库导入方案设计与技术选型》《题库与题目管理架构设计》《技术路线文档》中分散且冲突的数据模型，作为**唯一权威 schema**。其余文档中的 Prisma 草案以本文档为准，冲突处一律服从此处。
>
> 适用技术栈：Next.js（全栈）+ Prisma + MySQL 8.0。

---

## 0. 已拍板的全局决策

| 决策点 | 结论 | 来源冲突 |
|---|---|---|
| 主键类型 | **全库 `Int @id @default(autoincrement())`** | 解决 cuid(String) vs Int 冲突 |
| 身份模型 | **独立 `Student` + `Teacher` 两表**（统一 User 表示例作废） | 解决身份建模冲突；补齐 Teacher 表 |
| 一期题型 | **单选 / 多选 / 判断 / 填空**（不含简答 → 无需人工批改流程） | 解决 5 种 vs 4 种冲突 |
| 难度 | `enum Difficulty { EASY MEDIUM HARD }`，默认 `MEDIUM` | 解决 String vs Int(1-5) |
| options/answer 格式 | `options=[{key,text}]`、`answer` 一律数组化（含填空） | 解决两套 JSON 格式冲突 |
| 归属维度 | 一期平铺 nullable String；知识点二期升多对多 | 统一 textbook/unit/knowledgePoint |
| 题目去重键 | `@@unique([bankId, contentHash])`（库内去重，允许同题进多库） | 修正全局唯一冲突 |
| 密码存储 | 学生与教师**均存 bcrypt 哈希**，绝不存明文 | 教师密码补齐哈希 |

## 1. 枚举

```prisma
enum Difficulty {
  EASY
  MEDIUM
  HARD
}

enum QuestionType {
  SINGLE_CHOICE   // 单选
  MULTIPLE_CHOICE // 多选
  TRUE_FALSE      // 判断
  FILL_BLANK      // 填空（一期实现，自动判分见 §6）
  // SHORT_ANSWER 简答 —— 二期，需配人工批改流程
}

enum ExamType {
  EXAM      // 考试
  PRACTICE  // 练习
}

enum AttemptStatus {
  IN_PROGRESS // 进行中（支持断点续答）
  SUBMITTED   // 已提交
}
```

## 2. 身份与班级模块

### 2.1 Teacher（教师，独立表）

```prisma
model Teacher {
  id           Int      @id @default(autoincrement())
  username     String   @unique          // 登录账号，由超管导入
  passwordHash String                    // bcrypt 哈希，绝不存明文
  name         String
  createdAt    DateTime @default(now())

  classes       Class[]
  questionBanks QuestionBank[]
  exams         Exam[]
}
```

### 2.2 Student（学生身份，独立表）

```prisma
model Student {
  id           Int          @id @default(autoincrement())
  studentNo    String       @unique @db.VarChar(10) // 9-10位学号，全局唯一身份，String 防丢前导零
  name         String
  passwordHash String                                // 默认密码 wxls12345 的 bcrypt 哈希（整批只算一次）
  mustResetPwd Boolean      @default(true)            // 预留：是否仍用默认密码 / 强制改密
  deletedAt    DateTime?                              // 预留：软删除（一期不强制使用）
  createdAt    DateTime     @default(now())

  enrollments  Enrollment[]
  attempts     Attempt[]
  wrongQuestions WrongQuestion[]
}
```

### 2.3 Class（班级）

```prisma
model Class {
  id          Int          @id @default(autoincrement())
  name        String
  teacherId   Int                                    // 创建教师
  teacher     Teacher      @relation(fields: [teacherId], references: [id])
  createdAt   DateTime     @default(now())

  enrollments Enrollment[]
  examClasses ExamClass[]
}
```

### 2.4 Enrollment（学生↔班级成员关系中间表）

> 底层多对多；一期"一个学生只能在一个班"由**导入校验（应用层）**实现，不写死进 DB。

```prisma
model Enrollment {
  id        Int      @id @default(autoincrement())
  studentId Int
  classId   Int
  // role   String   @default("regular") // 预留：竞赛班用 regular/contest 区分
  student   Student  @relation(fields: [studentId], references: [id], onDelete: Cascade)
  class     Class    @relation(fields: [classId], references: [id], onDelete: Cascade)

  @@unique([studentId, classId]) // 同一人不在同一班重复登记
  @@index([classId])
}
```

## 3. 题库与题目模块（管理 / 编辑层）

### 3.1 QuestionBank（题库）

```prisma
model QuestionBank {
  id          Int        @id @default(autoincrement())
  name        String
  description String?    @db.Text
  subject     String?                      // 科目/课程（二期升级为独立课程表）
  createdBy   Int                          // 创建教师 id
  teacher     Teacher    @relation(fields: [createdBy], references: [id])
  createdAt   DateTime   @default(now())
  updatedAt   DateTime   @updatedAt

  questions   Question[]
}
```

### 3.2 Question（题目，合并版 —— 含导入字段）

> 单一归属：每题属于唯一题库。导入时 `bankId` 来自**当前题库页面上下文**（与学生导入的 classId 对称），不在 Excel 中。

```prisma
model Question {
  id            Int          @id @default(autoincrement())
  bankId        Int
  bank          QuestionBank @relation(fields: [bankId], references: [id], onDelete: Cascade)

  type          QuestionType
  stem          String       @db.Text       // 题干；填空用占位符标空（如 ____ 或 {{1}}）
  options       Json?                        // [{key:"A",text:"..."}]；判断/填空为 null
  answer        Json                         // 见 §6 答案编码（一律数组化）
  difficulty    Difficulty   @default(MEDIUM)
  analysis      String?      @db.Text

  // 归属维度（一期平铺冗余 String，二期 knowledgePoint 升多对多）
  textbook      String?
  unit          String?
  knowledgePoint String?

  // 导入 / 运维字段
  tags          String?                      // 预留：逗号分隔，后续升多对多
  score         Int?                         // 预留：刷题不用，权威分值在 ExamQuestion
  externalId    String?                      // 原始资料编号，支持"重导更新"
  contentHash   String                       // 去重：题型+归一化题干+答案 的 SHA1
  importBatchId String?                      // 导入批次，支持整批回滚

  createdAt     DateTime     @default(now())
  updatedAt     DateTime     @updatedAt

  knowledgePoints QuestionKnowledgePoint[]   // 二期

  @@unique([bankId, contentHash])            // 库内去重；允许同题进多个题库
  @@index([type])
  @@index([difficulty])
  @@index([importBatchId])
}
```

### 3.3 KnowledgePoint / QuestionKnowledgePoint（二期，建议一期先建表）

```prisma
model KnowledgePoint {

## 4. 考试 / 试卷模块（快照层）

### 4.1 Exam（考试 / 练习）

```prisma
model Exam {
  id               Int            @id @default(autoincrement())
  name             String
  type             ExamType
  allowRepeat      Boolean        @default(false)  // 是否可重复练习
  repeatLimit      Int?                            // 可重做次数（null=不限）
  deadline         DateTime?                       // 截止时间（可空）
  timeLimitSec     Int?                            // 单次作答时长上限（秒），null=不限时
  shuffleQuestions Boolean        @default(false)
  shuffleOptions   Boolean        @default(false)
  createdBy        Int
  teacher          Teacher        @relation(fields: [createdBy], references: [id])
  createdAt        DateTime       @default(now())
  deletedAt        DateTime?                       // 已有作答时软删除

  examQuestions    ExamQuestion[]
  classes          ExamClass[]
  attempts         Attempt[]
}
```

### 4.2 ExamQuestion（试卷题目快照）

```prisma
model ExamQuestion {
  id          Int          @id @default(autoincrement())
  examId      Int
  exam        Exam         @relation(fields: [examId], references: [id], onDelete: Cascade)
  sourceId    Int?                          // 溯源原题 id（弱引用，原题删除不影响卷面）
  order       Int
  score       Float                         // 卷面级分值（权威分值）
  // ↓ 组卷时从 Question 复制的快照字段 ↓
  type        QuestionType
  stem        String       @db.Text
  options     Json?
  answer      Json
  analysis    String?      @db.Text

  answerItems AnswerItem[]
}
```

### 4.3 ExamClass（考试↔班级）

```prisma
model ExamClass {
  examId  Int
  classId Int
  exam    Exam  @relation(fields: [examId], references: [id], onDelete: Cascade)
  class   Class @relation(fields: [classId], references: [id], onDelete: Cascade)
  @@id([examId, classId])
}
```

## 5. 作答 / 成绩模块

### 5.1 Attempt（作答记录，支持断点续答）

```prisma
model Attempt {
  id          Int           @id @default(autoincrement())
  examId      Int
  exam        Exam          @relation(fields: [examId], references: [id], onDelete: Cascade)
  studentId   Int
  student     Student       @relation(fields: [studentId], references: [id], onDelete: Restrict) // 有作答禁止硬删学生
  attemptNo   Int                           // 第几次作答，从 1 开始
  status      AttemptStatus @default(IN_PROGRESS)
  score       Float?
  elapsedSec  Int           @default(0)      // 已用时间（断点续答 / 考试计时）
  startedAt   DateTime      @default(now())
  lastSavedAt DateTime?                      // 最后一次草稿保存时间
  submittedAt DateTime?

  answerItems AnswerItem[]
  @@unique([examId, studentId, attemptNo])
}
```

> **断点续答**：进行中作答边做边写 `AnswerItem`（草稿态），`elapsedSec`/`lastSavedAt` 随之更新；学生中途退出再进入时恢复进度与已用时间。

### 5.2 AnswerItem（答题明细）

```prisma
model AnswerItem {
  id             Int          @id @default(autoincrement())
  attemptId      Int
  attempt        Attempt      @relation(fields: [attemptId], references: [id], onDelete: Cascade)
  examQuestionId Int
  examQuestion   ExamQuestion @relation(fields: [examQuestionId], references: [id], onDelete: Cascade)
  chosen         Json                       // 学生作答，结构同 answer（含填空数组）
  isCorrect      Boolean      @default(false)
  scoreGot       Float        @default(0)
  @@index([examQuestionId])
}
```

### 5.3 WrongQuestion（错题本 / 错题重做次数）

> 错题本身可由 `AnswerItem.isCorrect=false` 推导，但"重做次数"需独立持久化。一期按考试/练习分开记录。

```prisma
model WrongQuestion {
  id             Int      @id @default(autoincrement())
  studentId      Int
  student        Student  @relation(fields: [studentId], references: [id], onDelete: Cascade)
  examQuestionId Int                          // 指向快照，保证错题内容长期稳定
  redoCount      Int      @default(0)         // 重做次数
  lastResult     Boolean?                     // 最近一次重做是否正确
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt
  @@unique([studentId, examQuestionId])
}
```

## 6. JSON 字段约定（options / answer / chosen）

统一规则：`options` 为带稳定 key 的数组，`answer`/`chosen` 一律数组化，乱序只改展示不改 key。

| 题型 | options | answer | chosen 示例 |
|---|---|---|---|
| 单选 | `[{key:"A",text:"…"},…]` | `["A"]` | `["B"]` |
| 多选 | `[{key:"A",text:"…"},…]` | `["A","C"]` | `["A","B"]` |
| 判断 | `null` | `["T"]` / `["F"]` | `["T"]` |
| 填空 | `null` | `[["北京","京城"],["上海"]]`（每空一个数组，空内为可接受答案） | `["北京","上海"]` |

**填空自动判分（一期口径）**：

- 解析阶段：题干占位符个数 == answer 段数（`###` 段数），否则预检拦截。
- 判分：对每个空，把学生作答归一化（trim、全半角、可选忽略大小写）后，命中该空可接受答案集合（`||` 分隔）任一即该空正确。
- **整题判对错口径**：一期采用**全对才算对**（所有空命中 → `isCorrect=true`，得满分；否则 `isCorrect=false`，得 0 分）。如需"按空给分"再在二期放开 `scoreGot` 部分计分。

## 7. 级联删除与数据完整性

| 删除对象 | onDelete 行为 | 说明 |
|---|---|---|
| QuestionBank | 级联删 Question | 试卷有快照不受影响 |
| Question | ExamQuestion.sourceId 弱引用/SetNull | 仅丢溯源，卷面完好 |
| Exam（无作答） | 级联删 ExamQuestion / ExamClass | 安全 |
| Exam（已有作答） | 软删除（写 deletedAt） | 保护成绩 |
| Student（误导入纠正） | 硬删，级联清 Enrollment | 仅限刚导入未作答 |
| Student（已有作答） | `Attempt.studentId` 配 `onDelete: Restrict` 阻断硬删 → 走软删除 deletedAt | 防误删成绩 |
| Class | 仅删 Enrollment / ExamClass 关联 | 不删考试 |
| Attempt | 级联删 AnswerItem | |

## 8. 统计口径

- 平均分 / 分数段：基于 `Attempt.score`。
- 每题准确率：`AnswerItem` 按 `examQuestionId` 分组，正确数 / 总作答数。
- 选项选择率：`AnswerItem.chosen` 按 `examQuestionId` + 选项 key 分组（仅选择题适用；填空不统计选择率，仍统计准确率）。
- **可重复练习口径**：一期统一取**最近一次** `Attempt`（`attemptNo` 最大且 `status=SUBMITTED`），不做配置项。
