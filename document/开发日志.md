## [2026-06-30] S1 — 鉴权系统 ✅

### 完成内容
- **NextAuth 选型 v5（Auth.js，`next-auth@5.0.0-beta.31`）**：v4(4.24.14) 不支持 Next16/React19；v5 peerDeps 明确 `next ^16` + `react ^19`
- **split config 模式**：`src/auth.config.ts`（Edge 安全，仅 jwt/session 回调，不引入 Prisma）+ `src/auth.ts`（注入访问数据库的 Credentials Provider）
- `authorize` 双身份判分：学生查 `Student.studentNo`（过滤 `deletedAt`）；教师/超管查 `Teacher.username`；**admin 门户强制校验 `isAdmin`**；role 取自登录门户
- `src/proxy.ts` 路由隔离（Next16 已将 `middleware.ts` 改名为 `proxy.ts`）：未登录→本区登录页(带 callbackUrl)；跨角色→各自首页；已登录访登录页→回首页
- **服务端二次鉴权** `src/lib/auth-guard.ts` 的 `requireRole()`：满足 PRD §line113「middleware + handler 双重校验」，三个受保护页均接入
- 三套登录页（教师/超管 PC 卡片 / 学生移动毛玻璃）共用 `LoginForm`（Zod 校验、`signIn(redirect:false)`、安全 callbackUrl 回跳）
- 设计 token 落地 `globals.css`（Tailwind v4 `@theme`）+ `layout.tsx` 换 Inter 字体 + `SessionProvider`
- 安装 `zod`；类型增强 `src/types/next-auth.d.ts`

### 验收结果（全部通过）
- `tsc --noEmit` 0 错误；`next build` 0 警告（proxy 正确编译为 Edge）
- **18 项 curl 集成测试全过**：三角色登录成功 / 错误凭证被拒 / 普通教师走 admin 门户被拒 / 未登录跳本区登录 / 跨角色拦截回首页 / 同角色 200 放行 / 已登录(含异区)访登录页回首页
- 浏览器截图确认学生(移动)与教师(PC)登录页 Apple 观感渲染正确

### 遇到的问题及解决方案
- 问题：`declare module "next-auth/jwt"` 的 JWT 类型增强不生效（token 字段为 unknown）
  解决：`next-auth/jwt` 只是 `export * from "@auth/core/jwt"` 的纯再导出，须直接增强 `@auth/core/jwt`
- 问题：`next build` 报 `middleware` 约定已废弃
  解决：按 Next16 改名为 `src/proxy.ts`（功能等价，警告消除）
- 问题：`next start` 与 `output:standalone` 不完全兼容（仅自测期）
  解决：自测可用；正式部署按技术路线用 `node .next/standalone/server.js`

### 代码评审（多智能体对抗式审查，11 项 raise → 4 项确认并已修复）
- [medium] 受保护页缺服务端鉴权（单点依赖 proxy）→ 加 `requireRole()` 二次校验
- [low] callbackUrl 用 `startsWith` 无路径段边界 → 改 `=== homePath || startsWith(homePath+"/")`
- [low] 已登录用户访问异区登录页未重定向 → 登录页分支改为「任何已登录用户回各自首页」
- [low] 登录按钮 52px < design 规定 56px → 改 `h-14`，删冗余 inline style
- 已驳回 7 项（如 .env.example「v4 变量名」属误判：v5 实际兼容读取 NEXTAUTH_SECRET；时序侧信道枚举留待加限流时一并处理）

### 尚未解决 / 暂缓
